Arquitectura de Inferencia
y Flujo de Datos

Cómo entran los datos, cómo los procesa cada modelo internamente, y cómo se produce el ranking de cultivos recomendados para una parcela en Cundinamarca.

1. ¿Qué es la Inferencia?

La inferencia es el momento en que un agricultor dibuja su parcela en el mapa y el sistema le devuelve un ranking de cultivos recomendados. A diferencia del entrenamiento (que ocurre una sola vez con datos históricos), la inferencia ocurre en tiempo real cada vez que un usuario consulta.

El proceso completo —desde que el agricultor dibuja un polígono hasta que ve sus cultivos recomendados— toma menos de 2 segundos. Internamente ocurren cuatro grandes fases: extracción de datos del polígono, preprocesamiento, predicción por los modelos, y combinación del resultado final.

3. Fase 1 — Entrada de Datos (del polígono al vector)

El agricultor solo dibuja un polígono. Todo lo demás es automático. El sistema extrae las variables de las capas geoespaciales precalculadas y construye un único vector numérico que alimenta a los modelos.

1

El agricultor dibuja su parcela

En la interfaz web, el usuario dibuja un polígono sobre el mapa de Cundinamarca, o ingresa coordenadas. Esto produce un GeoJSON con la geometría de la parcela. No necesita ingresar ningún dato manualmente sobre suelo, clima o topografía.

2

Extracción geoespacial automática (Zonal Stats)

El sistema intersecta el polígono con las capas raster y vectoriales precargadas. Para cada variable, calcula la media ponderada por área de los píxeles que caen dentro del polígono. Esto produce un valor único por variable.

Se construye el vector de entrada

Al final de la extracción, se tiene un único vector numérico de 74+ valores que representa las condiciones agroecológicas exactas de esa parcela. Este vector es la «huella ambiental» de la finca.

4. Fase 2 — Preprocesamiento del Vector

Antes de entrar a los modelos, el vector se transforma:

A

Imputación de valores faltantes

Si alguna capa geoespacial no tiene cobertura para el polígono (ej: estación IDEAM lejana), se imputa con la mediana del municipio o con el valor de la capa global (SoilGrids, CHIRPS). XGBoost y LightGBM manejan NaN nativamente, pero RF y TabNet requieren imputación.

B

Normalización (Min-Max Scaling)

Para TabNet y LSTM, los features numéricos se normalizan al rango [0, 1] usando los mínimos y máximos del dataset de entrenamiento. Los modelos basados en árboles (RF, XGBoost, LightGBM) no requieren normalización porque deciden por umbrales, no por distancias.

C

Codificación de categóricos

Variables como piso_termico y vocacion_uso se codifican. LightGBM las recibe como categóricos nativos. Para los demás modelos se aplica label encoding (ordinal) o target encoding.

D

Reshape para LSTM

LSTM requiere datos en formato 3D: (batch, timesteps, features). Los 12 valores mensuales de precipitación y los 6 estadísticos temporales de NDVI se reorganizan como una secuencia de 12 timesteps × N features por timestep. El resto de variables estáticas se concatenan al final.

Dos formatos de datos según el modelo
# FORMATO A: Tabular plano → para RF, XGBoost, LightGBM, TabNet X_tabular = np.array([[19.5, 26.2, 13.8, ..., 0.62, 0.95]]) # shape: (1, 74) # FORMATO B: Secuencial → para LSTM # Reorganizar los 12 meses como secuencia temporal X_temporal = np.array([[[78.0, 0.45, -12.3], # enero: precip, ndvi, vv [92.0, 0.52, -11.8], # febrero [115.0, 0.61, -13.1], # marzo ... # hasta diciembre [60.0, 0.38, -14.2]]]) # shape: (1, 12, 3) # 1 parcela, 12 meses, 3 features por mes

5. Fase 3 — Cómo Procesa Cada Modelo los Datos

5.1 Random Forest — Votación de 300 árboles

RF recibe el vector tabular (1, 74) y lo pasa por 300 árboles de decisión independientes. Cada árbol fue entrenado con un subconjunto aleatorio de filas y columnas del dataset. Cada árbol emite un voto: «este vector se parece más a parcelas donde se siembra Papa» (o Maíz, o Arveja, etc.).

📥
Entrada
Vector tabular plano de la parcela
(1, 74)
🌳
300 Árboles de Decisión (paralelos)
Cada árbol recorre sus nodos preguntando: «¿pH < 5.5?» → izquierda; «¿elevación > 2000?» → derecha. Al llegar a una hoja, emite su voto de clase.
300 votos
🗳️
Agregación (Votación / Promedio de probabilidades)
Se cuentan los votos: si 180/300 árboles votaron «Papa», 70 «Arveja», 50 «Maíz» → P(Papa)=0.60, P(Arveja)=0.23, P(Maíz)=0.17
(1, N_clases)
📤
Salida: vector de probabilidades
[0.60, 0.23, 0.17, 0.00, ...] — una probabilidad por cada cultivo del catálogo
(1, N_clases)

5.2 XGBoost — Corrección secuencial de errores

XGBoost también recibe (1, 74), pero en lugar de árboles independientes, sus árboles trabajan en secuencia. El primer árbol hace una predicción aproximada. El segundo árbol se enfoca en corregir los errores del primero. El tercero corrige los errores del segundo, y así sucesivamente por 200-500 iteraciones.

📥
Entrada
Vector tabular plano
(1, 74)
🌳→🌳→🌳
Árboles secuenciales (boosting)
Árbol 1 predice. Árbol 2 predice el error del 1. Árbol 3 predice el error residual. Cada uno agrega una corrección pequeña (learning_rate × predicción).
200-500 árboles
🛡️
Regularización L1/L2
Penaliza árboles demasiado complejos para evitar overfitting. Controla que el modelo no memorice el dataset de entrenamiento.
λ × Ω(f)
📤
Salida: Suma acumulada → softmax → probabilidades
[0.58, 0.25, 0.12, 0.05, ...] — probabilidad por cultivo
(1, N_clases)

5.3 LightGBM — Histogramas para velocidad

LightGBM funciona igual que XGBoost conceptualmente (boosting secuencial), pero acelera el proceso creando histogramas de los features en lugar de evaluar cada punto de corte posible. Además, con GOSS muestrea las instancias más informativas y con EFB agrupa features correlacionados.

Diferencia práctica: LightGBM produce resultados comparables a XGBoost pero entrena 5-10× más rápido en datasets grandes. Su salida es idéntica en formato: un vector de probabilidades (1, N_clases).

5.4 TabNet — Atención secuencial sobre features tabulares

TabNet es una red neuronal diseñada para datos tabulares. En lugar de procesar todos los features de golpe, usa un mecanismo de atención secuencial: en cada «paso de decisión» selecciona qué features son relevantes y procesa solo esos. Esto lo hace interpretable: podemos saber qué features usó en cada paso.

📥
Entrada
Vector tabular normalizado [0,1]
(1, 74)
🔍
Batch Normalization inicial
Estabiliza los valores de entrada para que la red aprenda más eficientemente
(1, 74)
👁️
Paso de Decisión 1 — Máscara de Atención
La red decide: «para esta parcela, pH y elevación son los más importantes ahora». Crea una máscara sparse que pondera los features.
Máscara (1, 74) → sparse
⚙️
Transformación FC + GLU
Los features seleccionados pasan por capas fully-connected con activación GLU (Gated Linear Unit). Se produce una representación parcial.
(1, n_d)
👁️
Pasos 2, 3, ... N_steps — Atención acumulada
Se repite: nueva máscara de atención (ahora priorizando N, P, K), nueva transformación. Cada paso agrega información complementaria. El coeficiente γ controla la reutilización de features.
N_steps = 3-7
∑
Agregación de pasos + Fully Connected final
Las representaciones de todos los pasos se suman. Una capa FC final mapea a las clases de cultivo.
(1, N_clases)
📤
Softmax → Probabilidades
[0.55, 0.22, 0.15, 0.08, ...]
(1, N_clases)

5.5 LSTM — Procesamiento de secuencias temporales

LSTM recibe las variables mensuales como una secuencia de 12 pasos (enero a diciembre). En cada paso temporal, la red decide qué información recordar, qué olvidar y qué pasar al siguiente mes. Esto le permite entender patrones como: «lluvia fuerte en abril seguida de sequía en julio afecta diferente que lluvia distribuida todo el año».

📥
Entrada temporal
Secuencia de 12 meses, cada uno con precipitación, NDVI, temperatura, humedad y backscatter
(1, 12, 5)
🚪
Capa LSTM 1 (96 unidades)
Forget Gate: σ(W_f · [h_{t-1}, x_t] + b_f) — decide qué olvidar del mes anterior. Input Gate: σ(W_i · [h_{t-1}, x_t]) — decide qué nueva información guardar. Output Gate: σ(W_o · [h_{t-1}, x_t]) — decide qué transmitir al mes siguiente.
(1, 12, 96)
🔄
Capa LSTM 2 (96 unidades)
Segunda capa que aprende patrones de orden superior sobre la representación de la primera capa.
(1, 96) ← solo último timestep
🔗
Concatenación con features estáticos
La salida del LSTM (96 valores que resumen el patrón temporal) se concatena con los features no temporales: pH, elevación, pendiente, etc.
(1, 96 + 30) = (1, 126)
📤
Fully Connected + Softmax → Probabilidades
[0.52, 0.28, 0.13, 0.07, ...]
(1, N_clases)

6. Fase 4 — El Meta-Learner Combina Todo

Cada modelo individual produce su propio vector de probabilidades. El meta-learner recibe todos estos vectores concatenados y aprende a combinarlos de forma óptima.

Cómo se construye la entrada del meta-learner
# 5 modelos, cada uno produce N_clases probabilidades # Para un catálogo de 40 cultivos: prob_RF = [0.60, 0.23, 0.17, 0.00, ..., 0.00] # (1, 40) prob_XGB = [0.58, 0.25, 0.12, 0.05, ..., 0.00] # (1, 40) prob_LGBM = [0.55, 0.27, 0.13, 0.05, ..., 0.00] # (1, 40) prob_TabNet = [0.52, 0.22, 0.18, 0.08, ..., 0.00] # (1, 40) prob_LSTM = [0.48, 0.30, 0.15, 0.07, ..., 0.00] # (1, 40) # Concatenar todos X_meta = np.concatenate([prob_RF, prob_XGB, prob_LGBM, prob_TabNet, prob_LSTM]) # Resultado: (1, 200) — 5 modelos × 40 cultivos = 200 features para el meta-learner # El meta-learner (Logistic Regression o SVC lineal) produce la predicción final prob_final = meta_learner.predict_proba(X_meta) # (1, 40) # Ranking: ordenar de mayor a menor probabilidad ranking = np.argsort(prob_final[0])[::-1] # Resultado: [Papa, Maíz, Arveja, Habichuela, Tomate, ...]

El meta-learner no es una red neuronal profunda. Es un modelo simple (Logistic Regression o SVC con kernel lineal) que aprende pesos como: «cuando RF dice Papa con >50% y LSTM también, confiar mucho; cuando solo TabNet dice Cacao pero los demás no, confiar menos». Estos pesos se aprendieron durante el entrenamiento con cross-validation.

7. Fase 5 — Salida al Agricultor

El resultado que ve el agricultor no es una tabla de probabilidades. Es un ranking ordenado y comprensible con explicaciones de por qué se recomienda cada cultivo.

Tiempo total de inferencia: Extracción geoespacial ~0.5s + Preprocesamiento ~0.1s + RF (0.06s) + XGBoost (0.07s) + LightGBM (0.07s) + TabNet (0.15s) + LSTM (0.12s) + Meta-learner (0.01s) + XAI/SHAP (~0.3s) = ~1.4 segundos totales. Los 5 modelos individuales se ejecutan en paralelo, reduciendo el tiempo efectivo.
