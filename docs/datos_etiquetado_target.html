1. Resumen de las Tres Fuentes de Etiquetado

El modelo «¿Qué Sembrar?» requiere etiquetas que indiquen qué cultivos se han sembrado con éxito en cada ubicación. Se combinan tres fuentes complementarias con diferente resolución espacial, temporal y temática.

2. Fuente 1 — Evaluaciones Agropecuarias Municipales (EVA)

2.1 ¿Qué es la EVA?

Las EVA son la operación estadística que recopila información sobre área, producción y rendimiento de la actividad agropecuaria en los 1.104 municipios de Colombia. Cubren cerca de 200 productos agrícolas. Existen dos datasets principales según el periodo de coordinación institucional:

# Python — Descargar EVA filtrada para Cundinamarca import pandas as pd # Dataset histórico 2007-2018 url_hist = "https://www.datos.gov.co/resource/2pnw-mmge.json?$where=c_d_dep='25'&$limit=50000" df_hist = pd.read_json(url_hist) # Dataset UPRA 2019-2024 url_upra = "https://www.datos.gov.co/resource/uejq-wxrr.json?$where=codigo_dane_departamento='25'&$limit=50000" df_upra = pd.read_json(url_upra) # Concatenar ambos periodos df_eva = pd.concat([df_hist, df_upra], ignore_index=True) print(f"Total registros Cundinamarca: {len(df_eva)}")

3. Fuente 2 — Monitoreo Satelital de Cultivos (UPRA)

3.1 ¿Qué es el Monitoreo Satelital?

Sistema implementado por la UPRA que utiliza imágenes Sentinel-2 y Planet, procesadas con algoritmos de Random Forest en la nube, para delimitar polígonos de áreas productivas de cultivos específicos. Cada polígono indica dónde se cultivó un producto en un semestre determinado, a escala de parcela (1:25.000).

3.4 Descarga Programática con Python

# Python — Descargar polígonos de papa en Cundinamarca desde ArcGIS REST import requests import geopandas as gpd base_url = "https://geoservicios.upra.gov.co/arcgis/rest/services/" service = "MonitoreoCultivos/cambios_papa_2022_s1_s2/MapServer/0/query" params = { "where": "cod_depart = '25'", # Cundinamarca "outFields": "*", "f": "geojson", "returnGeometry": "true", "resultRecordCount": "5000" } r = requests.get(base_url + service, params=params) gdf_papa = gpd.GeoDataFrame.from_features(r.json()["features"]) print(f"Polígonos de papa en Cundinamarca: {len(gdf_papa)}")

4. Fuente 3 — Zonificación de Aptitud (SIPRA)

4.1 ¿Qué es la Zonificación de Aptitud?

Son mapas elaborados por la UPRA que clasifican el territorio nacional según la aptitud para cada cultivo específico, evaluando entre 20 y 30 criterios que incluyen clima, suelo, infraestructura, logística, mercado y restricciones legales. Cada punto del territorio recibe una de cinco categorías: Aptitud Alta, Media, Baja, No Apta, o Exclusiones Legales.

El SIPRA cuenta con 65 capas nacionales y 260 departamentales. Para Cundinamarca existen zonificaciones específicas de aguacate Lorena, arroz de riego, arveja, hortalizas de clima frío, zanahoria, papaya Tainung y mora Castilla, entre otras.

5. Estrategia de Etiquetado de Datos

El proceso de etiquetado combina las tres fuentes para construir una variable objetivo multiclase robusta. Se proponen dos enfoques complementarios según la escala de análisis.

5.1 Enfoque A — Etiquetado a Nivel Municipal (basado en EVA)

Este enfoque asigna a cada municipio de Cundinamarca el conjunto de cultivos viables y su rendimiento histórico.

1

Filtrar EVA para Cundinamarca

Seleccionar todos los registros con CÓD. DEP. = 25 del periodo 2007–2024. Esto genera un panel municipio × cultivo × semestre.

2

Calcular indicadores de éxito por municipio-cultivo

Para cada par (municipio, cultivo), calcular:

frecuencia_siembra = número de semestres/años en que se sembró ese cultivo en ese municipio (persistencia temporal).

rendimiento_promedio = media del rendimiento (t/ha) a lo largo de los años.

area_promedio = media del área sembrada.

ratio_cosecha = área cosechada / área sembrada (indicador de éxito del ciclo).

3

Definir la etiqueta «cultivo_recomendado»

Para cada municipio, crear un ranking de cultivos ordenado por un score compuesto:

# Score de aptitud revelada score = (w1 × frecuencia_normalizada + w2 × rendimiento_normalizado + w3 × area_normalizada + w4 × ratio_cosecha) # Pesos sugeridos (ajustables vía validación cruzada) w1 = 0.30 # Persistencia temporal w2 = 0.35 # Productividad w3 = 0.20 # Escala de adopción w4 = 0.15 # Éxito del ciclo
4

Asignar etiquetas a píxeles

Cada píxel dentro del polígono municipal hereda el ranking de cultivos de ese municipio. Los features de la vista minable (clima, suelo, satélite, topografía) corresponden al valor del píxel; la etiqueta proviene del municipio.

Limitación: Este enfoque asume homogeneidad intra-municipal. Un municipio con variación altitudinal grande (ej: Fusagasugá con zonas a 500 m y 2.000 m) tendrá la misma etiqueta para píxeles muy diferentes. El Enfoque B mitiga este problema.

5.2 Enfoque B — Etiquetado a Nivel de Parcela (basado en Monitoreo + SIPRA)

Este enfoque utiliza los polígonos georreferenciados del monitoreo satelital como etiquetas de alta precisión espacial.

1

Descargar polígonos de monitoreo para Cundinamarca

Obtener los polígonos de papa, maíz, arroz, cacao, plátano, caña panelera y pastos desde los geoservicios UPRA. Filtrar por cod_depart = '25'.

2

Etiquetar píxeles con el cultivo del polígono

Cada píxel (10 m) que cae dentro de un polígono de monitoreo recibe la etiqueta del campo cadena (ej: «Papa», «Maíz», «Arroz»). Los features del píxel provienen de las capas climáticas, edafológicas, satelitales y topográficas para el periodo correspondiente.

3

Complementar con zonificación SIPRA

Para píxeles fuera de polígonos de monitoreo, intersectar con las capas de aptitud SIPRA. Un píxel clasificado como «Aptitud Alta» para papa por el SIPRA puede recibir una etiqueta suave (probabilística) o usarse como dato de validación.

4

Generar muestras negativas

Las zonas clasificadas como «No Apta» o «Exclusiones Legales» por el SIPRA constituyen muestras negativas explícitas. Incluirlas como clase «NO_APTO» mejora la capacidad del modelo para rechazar recomendaciones en zonas inadecuadas.

Ventaja del Enfoque B: Cada muestra de entrenamiento tiene correspondencia espacial exacta entre los features ambientales y la etiqueta de cultivo. Elimina el supuesto de homogeneidad municipal. Sin embargo, solo cubre 7 cultivos y el periodo 2021-2023.

aptitud_sipraCategóricoSIPRACategoría de aptitud oficial (Alta/Media/Baja/No Apta/Exclusión)
fuente_etiquetaStringMetadata«monitoreo_parcela», «eva_municipal» o «sipra_aptitud» — trazabilidad de la etiqueta
confianzaFloat [0-1]CompuestoNivel de confianza en la etiqueta: 1.0 para monitoreo de parcela, 0.7 para EVA consistente, 0.5 para EVA con baja frecuencia

Tipo de problema de ML: El modelo se formula como clasificación multiclase con ranking. Para cada píxel/parcela de entrada, el modelo predice las probabilidades de éxito para cada cultivo del catálogo. El output es un ranking de cultivos ordenado por probabilidad descendente. El rendimiento esperado (t/ha) de la EVA sirve como metadato complementario para presentar al agricultor, no como target de regresión.
